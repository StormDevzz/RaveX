## 1. Что такое Lua-аддон?

**Lua-аддон** — это `.lua` скрипт, который RaveX загружает через свой встроенный Lua-движок. В отличие от Java (JAR) или C++ (DLL/SO) аддонов, Lua-аддоны не требуют компиляции — просто напиши скрипт и положи его в папку аддонов.

### Lua vs Java vs C++

| Аспект | Lua | Java | C++ |
|--------|-----|------|-----|
| Сложность | Очень низкая | Низкая | Высокая |
| Нужна компиляция | Нет | Да (JAR) | Да (DLL/SO) |
| Производительность | Средняя | Средняя | Высокая |
| Доступ к API системы | Ограничен (sandbox) | Ограничен | Полный |
| Горячая перезагрузка | Да | Нет | Нет |
| Размер файла | ~1 KB | ~2 KB JAR | ~50-500 KB DLL/SO |

### Когда использовать Lua?

- **Быстрое прототипирование**: тестируй идеи без компиляции
- **Простые модули**: переключатели, чат-команды, твики GUI
- **Конфигурация**: динамические скрипты для чтения/записи настроек
- **Событийная логика**: реагируй на игровые события минимумом кода
- **Пользовательские скрипты**: дай пользователям менять поведение без C++/Java

## 2. Как работают Lua-аддоны

### Архитектура

```
┌─────────────────────────────────────────────┐
│                  RaveX Core                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ Java     │  │ C++      │  │ Lua      │   │
│  │ Аддоны   │  │ Аддоны   │  │ Движок   │   │
│  │ (JAR)    │  │ (DLL/SO) │  │ (C API)  │   │
│  └──────────┘  └──────────┘  └─────┬────┘   │
│                                    │         │
│                           ┌────────▼──────┐  │
│                           │ Lua-скрипты   │  │
│                           │ addons/lua/   │  │
│                           └───────────────┘  │
└─────────────────────────────────────────────┘
```

### Жизненный цикл

1. RaveX запускается → C++ Lua-подсистема инициализируется
2. Сканирует папку `addons/lua/` на `.lua` файлы
3. Для каждого файла:
   - Создаёт sandbox-окружение
   - Загружает и выполняет скрипт
   - Вызывает `getName()` и `getVersion()` для метаданных
   - Вызывает `onLoad(ctx)` — аддон активен
4. Во время игры:
   - Вызывает `onTick()` каждый игровой тик
   - Вызывает `onEvent(eventName)` при событиях
5. При завершении:
   - Вызывает `onUnload()` для каждого аддона
   - Уничтожает Lua-окружение

### Sandbox (песочница)

Для безопасности Lua-скрипты работают в изолированном окружении:
- `os.execute`, `os.exit`, `io.*` заблокированы
- `dofile`, `loadfile`, `require` заблокированы
- Доступны только API-функции RaveX и безопасные стандартные библиотеки Lua

## 3. 01 Минимальный аддон

**Расположение:** `01_minimal/main.lua`

Самый простой Lua-аддон. Каждый аддон обязан определить эти функции:

```lua
function getName()
    return "MyFirstAddon"
end

function getVersion()
    return "1.4.1"
end

function onLoad(ctx)
    logInfo("Привет из Lua!")
end

function onUnload()
    logInfo("Пока из Lua!")
end
```

### Что происходит

1. RaveX загружает `main.lua` из `addons/lua/MyFirstAddon/`
2. Вызывает `getName()` → "MyFirstAddon"
3. Вызывает `getVersion()` → "1.4.1"
4. Вызывает `onLoad(ctx)` → логирует "Привет из Lua!"
5. Аддон запущен

## 4. 02 Полнофункциональный аддон

**Расположение:** `02_features/main.lua`

Демонстрирует конфиг, модули, события, тики и платформенные ветки.

### Конфиг

```lua
-- Прочитать значение (с значением по умолчанию)
local enabled = configGet("enabled", "false")

-- Записать значение
configSet("last_loaded", "MyAddon v1.4.1")
```

Значения конфига сохраняются между сессиями игры.

### Модули

```lua
-- Зарегистрировать переключаемый модуль в GUI
registerModule("MyModule", "Описание моего модуля")

-- Проверить, включил ли его пользователь
if isModuleEnabled("MyModule") then
    logInfo("Модуль активен!")
end
```

### События

```lua
function onEvent(eventName)
    if eventName == "world_load" then
        logInfo("Мир загружен!")
    end
end
```

### Тики

```lua
local counter = 0
function onTick()
    counter = counter + 1
    if counter % 100 == 0 then
        logInfo("Тик #" .. counter)
    end
end
```

### Платформенные ветки

```lua
if getPlatform() == "windows" then
    -- Специфичная для Windows логика
else
    -- Специфичная для Linux логика
end
```

## 5. Справочник API

### Логирование

| Функция | Описание |
|---------|----------|
| `logInfo(msg)` | Логирует информационное сообщение |
| `logWarn(msg)` | Логирует предупреждение |
| `logError(msg)` | Логирует ошибку |

### Конфиг

| Функция | Описание |
|---------|----------|
| `configGet(key, default)` | Получить значение конфига (строка) |
| `configSet(key, value)` | Установить значение конфига (строка) |

### Модули

| Функция | Описание |
|---------|----------|
| `registerModule(name, description)` | Зарегистрировать модуль в GUI |
| `isModuleEnabled(name)` | Проверить, включён ли модуль |

### Система

| Функция | Описание |
|---------|----------|
| `getPlatform()` | Возвращает "windows" или "linux" |
| `getApiVersion()` | Возвращает версию API RaveX |
| `getAddonDir()` | Возвращает путь к папке addons/lua |

### Колбэки аддона (определи в скрипте)

| Функция | Обязательна | Описание |
|---------|-------------|----------|
| `getName()` | Да | Возвращает уникальное имя аддона |
| `getVersion()` | Да | Возвращает версию (semver) |
| `onLoad(ctx)` | Да | Вызывается при загрузке |
| `onUnload()` | Да | Вызывается при выгрузке |
| `onTick()` | Нет | Вызывается каждый тик |
| `onEvent(name)` | Нет | Вызывается при событиях |
| `getAuthor()` | Нет | Возвращает имя автора |
| `getDescription()` | Нет | Возвращает описание аддона |

## 6. Установка

### Windows

```cmd
copy 01_minimal\main.lua "%USERPROFILE%\.minecraft\ravex\addons\lua\"
```

Или создай папку для своего аддона:

```cmd
mkdir "%USERPROFILE%\.minecraft\ravex\addons\lua\MyAddon"
copy main.lua "%USERPROFILE%\.minecraft\ravex\addons\lua\MyAddon\"
```

### Linux

```bash
cp 01_minimal/main.lua ~/.minecraft/ravex/addons/lua/
mkdir -p ~/.minecraft/ravex/addons/lua/MyAddon
cp main.lua ~/.minecraft/ravex/addons/lua/MyAddon/
```

### Структура папок

```
~/.minecraft/ravex/addons/lua/
├── MyFirstAddon/
│   └── main.lua
├── MySecondAddon/
│   └── main.lua
└── standalone.lua       # можно и прямо в lua/
```

RaveX рекурсивно сканирует папку `lua/` и загружает все `.lua` файлы.
